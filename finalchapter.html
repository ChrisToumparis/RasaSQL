Πλήρης Εκπαίδευση SQL

Όλα τα κεφάλαια σε μία σελίδα για εύκολη ανάγνωση και αναφορά.

Εισαγωγή

1.1 Τι είναι η SQL;

Η SQL (Structured Query Language) είναι η γλώσσα δομημένων ερωτημάτων που χρησιμοποιείται για τη διαχείριση και τον χειρισμό σχεσιακών βάσεων δεδομένων. Αποτελεί παγκόσμιο πρότυπο και είναι μία από τις πιο σημαντικές γλώσσες στον τομέα της πληροφορικής.

💡 Γιατί SQL; Η SQL είναι απαραίτητη για την ανάκτηση, τροποποίηση και διαχείριση δεδομένων σε σχεσιακές βάσεις δεδομένων όπως η MySQL, PostgreSQL, Oracle, SQL Server και άλλες.

Βασικά Χαρακτηριστικά της SQL:

  • 📊 Ανάκτηση Δεδομένων - Με ερωτήματα SELECT
  • ✏️ Τροποποίηση Δεδομένων - INSERT, UPDATE, DELETE
  • 🏗️ Ορισμός Δομής - CREATE, ALTER, DROP
  • 🔒 Έλεγχος Πρόσβασης - GRANT, REVOKE

Η SQL και η MySQL

2.1 Η SQL

Η SQL (Structured Query Language) αποτελεί ένα παγκόσμιο πρότυπο για τον χειρισμό Σχεσιακών Βάσεων Δεδομένων. Αρχικά ονομαζόταν SEQUEL και υλοποιήθηκε για το System R της IBM.

📚 Ιστορία Η πρώτη τυποποιημένη έκδοση (SQL1) δημιουργήθηκε το 1986. Ακολούθησε η SQL2 (1992) και η SQL3/SQL-99 με αντικειμενοστραφή στοιχεία.

2.2 Η MySQL

Η MySQL είναι ένα από τα πιο δημοφιλή συστήματα διαχείρισης βάσεων δεδομένων. Βασικά χαρακτηριστικά:

  • ✅ Open Source (ανοιχτού κώδικα)
  • ⚡ Γρήγορη και αξιόπιστη
  • 🌐 Cross-platform (Windows, Linux, Mac)
  • 📖 Εξαιρετική τεκμηρίωση

2.3 Πίνακες

Οι πίνακες (tables) είναι οι βασικές δομές αποθήκευσης δεδομένων. Κάθε πίνακας αποτελείται από:

  • 📝 Γραμμές (Rows) - Εγγραφές/Records
  • 📊 Στήλες (Columns) - Πεδία/Fields

2.4 Τύποι Δεδομένων

Ερωτήματα Επιλογής

3.1 Η Εντολή SELECT

Η SELECT είναι η βασικότερη εντολή της SQL για ανάκτηση δεδομένων.

-- Επιλογή όλων των πεδίων SELECT * FROM City; -- Επιλογή συγκεκριμένων πεδίων SELECT Name, Population FROM City;

3.2 WHERE - Φιλτράρισμα Δεδομένων

Η πρόταση WHERE χρησιμοποιείται για φιλτράρισμα εγγραφών:

SELECT * FROM City WHERE Population > 1000000;

Τελεστές Σύγκρισης:

  • = Ίσο
  • > Μεγαλύτερο
  • < Μικρότερο
  • >= Μεγαλύτερο ή ίσο
  • <= Μικρότερο ή ίσο
  • <> ή != Διάφορο

3.3 ORDER BY - Ταξινόμηση

-- Αύξουσα (ASC - default) SELECT Name FROM City ORDER BY Population ASC; -- Φθίνουσα (DESC) SELECT Name FROM City ORDER BY Population DESC;

3.4 LIKE - Pattern Matching

Για αναζήτηση με μοτίβα χρησιμοποιούμε LIKE:

-- Πόλεις που αρχίζουν με 'A' SELECT Name FROM City WHERE Name LIKE 'A%'; -- Πόλεις που τελειώνουν σε 'ς' SELECT Name FROM City WHERE Name LIKE '%ς'; -- Όνομα περιέχει 'θην' SELECT Name FROM City WHERE Name LIKE '%θην%';

3.5 DISTINCT - Μοναδικές Τιμές

-- Όλες οι ηπείρους (χωρίς επαναλήψεις) SELECT DISTINCT Continent FROM Country;

3.6 LIMIT - Περιορισμός Αποτελεσμάτων

-- Μόνο τις 10 πρώτες εγγραφές SELECT * FROM City LIMIT 10; -- Από τη 20η έως τη 30η εγγραφή SELECT * FROM City LIMIT 20, 10;

Συνδυαστικές Συναρτήσεις

4.1 COUNT - Καταμέτρηση Εγγραφών

Η συνάρτηση COUNT() μετράει τον αριθμό των εγγραφών:

-- Πλήθος πόλεων SELECT COUNT(*) FROM City; -- Πλήθος μοναδικών χωρών SELECT COUNT(DISTINCT CountryCode) FROM City;

4.2 SUM και AVG - Άθροισμα & Μέσος Όρος

-- Συνολικός πληθυσμός SELECT SUM(Population) FROM Country; -- Μέσος πληθυσμός SELECT AVG(Population) FROM Country;

4.3 MIN και MAX

-- Μικρότερος και μεγαλύτερος πληθυσμός SELECT MIN(Population), MAX(Population) FROM City;

4.4 GROUP BY - Ομαδοποίηση

Ομαδοποιούμε δεδομένα για στατιστικά ανά κατηγορία:

-- Πλήθος χωρών ανά ήπειρο SELECT Continent, COUNT(*) FROM Country GROUP BY Continent; -- Μέσος πληθυσμός ανά ήπειρο SELECT Continent, AVG(Population) FROM Country GROUP BY Continent;

4.5 HAVING - Φιλτράρισμα Ομάδων

Το HAVING λειτουργεί σαν WHERE αλλά για ομάδες:

-- Ηπείρους με >10 χώρες SELECT Continent, COUNT(*) FROM Country GROUP BY Continent HAVING COUNT(*) > 10;
📊 WHERE vs HAVING WHERE = φιλτράρει εγγραφές ΠΡΙΝ το GROUP BY
HAVING = φιλτράρει ομάδες ΜΕΤΑ το GROUP BY

Ορισμός & Διαχείριση Δεδομένων

5.1 CREATE DATABASE - Δημιουργία Βάσης

CREATE DATABASE mydb;

5.2 CREATE TABLE - Δημιουργία Πίνακα

CREATE TABLE students ( id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50) NOT NULL, age INT, email VARCHAR(100) );

5.3 PRIMARY KEY - Πρωτεύον Κλειδί

Το PRIMARY KEY αναγνωρίζει μοναδικά κάθε εγγραφή:

  • ✅ Μοναδική τιμή για κάθε εγγραφή
  • ✅ Δεν επιτρέπει NULL
  • ✅ Ένα μόνο PRIMARY KEY ανά πίνακα

5.4 FOREIGN KEY - Ξένο Κλειδί

CREATE TABLE grades ( id INT PRIMARY KEY, student_id INT, grade FLOAT, FOREIGN KEY (student_id) REFERENCES students(id) );
🔗 Foreign Key Συνδέει δύο πίνακες και διασφαλίζει την ακεραιότητα των δεδομένων.

5.5 ALTER TABLE - Τροποποίηση Πίνακα

-- Προσθήκη πεδίου ALTER TABLE students ADD phone VARCHAR(15); -- Διαγραφή πεδίου ALTER TABLE students DROP COLUMN phone; -- Τροποποίηση πεδίου ALTER TABLE students MODIFY age SMALLINT;

5.6 DROP TABLE - Διαγραφή Πίνακα

DROP TABLE students;
⚠️ ΠΡΟΣΟΧΗ Η DROP διαγράφει ΜΟΝΙΜΑ τον πίνακα και όλα τα δεδομένα του!

Χειρισμός Δεδομένων

6.1 INSERT - Εισαγωγή Δεδομένων

INSERT INTO students (name, age) VALUES ('Νίκος', 22);

6.2 UPDATE - Ενημέρωση

UPDATE students SET age = 23 WHERE name = 'Νίκος';

6.3 DELETE - Διαγραφή

DELETE FROM students WHERE age < 18;
⚠️ ΠΡΟΣΟΧΗΧωρίς WHERE, θα διαγραφούν ΟΛΕΣ οι εγγραφές!

Ένωση Πινάκων

7.1 INNER JOIN

SELECT City.Name, Country.Name FROM City INNER JOIN Country ON City.CountryCode = Country.Code;

7.2 LEFT JOIN

SELECT * FROM Country LEFT JOIN City ON Country.Code = City.CountryCode;

7.3 RIGHT JOIN

Όπως το LEFT αλλά επιστρέφει όλες τις εγγραφές από τον δεξί πίνακα.

Υποερωτήματα

8.1 Απλό Υποερώτημα

SELECT Name FROM Country WHERE Population > (SELECT AVG(Population) FROM Country);

8.2 IN με Υποερώτημα

SELECT Name FROM City WHERE CountryCode IN ( SELECT Code FROM Country WHERE Continent = 'Europe' );

8.3 EXISTS

SELECT Name FROM Country WHERE EXISTS (SELECT * FROM City WHERE CountryCode = Country.Code);

Όψεις (Views)

9.1 CREATE VIEW

CREATE VIEW european_cities AS SELECT * FROM City WHERE CountryCode IN ( SELECT Code FROM Country WHERE Continent = 'Europe' );

9.2 Χρήση Views

SELECT * FROM european_cities;

9.3 DROP VIEW

DROP VIEW european_cities;
